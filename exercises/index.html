Exercises

Simple Component (1/5)

  1. Let's assume we're in the app directory.
  2. Create the directory books which will contain all book components' files.
  3. Create the Book class (in the file book/book.ts) having the id, author and title properties (including a constructor as well as setters and getters). Don't forget to export the class so that it is available for other classes to import.

Simple Component (2/5)

  1. Create the BookDetailsComponent class (in the file book/book-details.component.ts) having the currentBook public property of type Book.
  2. Initialize the currentBook property with an empty Book object in the constructor.
  3. Take part in the OnInit lifecycle hook letting the class to implement the OnInit interface and its ngOnInit() method. It then sets the author and title properties of the currentBook.

Simple Component (3/5)

  1. Add the @Component decorator to the BookDetailsComponent class. In the object passed to the decorator set the selector property to book-details.
  2. Create the template (in the file book/book-details.component.html) containing two block elements (e.g. <div>s) each of which contains a label and a value of the author and title respectively. Use the interpolation {{}} for showing the values.

Simple Component (4/5)

  1. Set the @Component decorator's property template to require('./book-details.component.html!text').
  2. Add the BookDetailsComponent to the AppModule's declarations (in the @NgModule decorator).
  3. Add the <book-details></book-details> element to the AppComponent's template.

Simple Component (5/5)

  1. Create the file book/book-details.component.less.
  2. Add some component's styling to the file, e.g.:
    • make the host element (<book-details>) a block element
    • make the labels' font weight bold
    Make use of the :host pseudo selector.
  3. Wire the styling to the component's metadata: set the @Component decorator's property styles to [require('./book-details.component.css!text')].
